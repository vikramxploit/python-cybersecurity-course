💥 Why Errors Happen

Every program encounters unexpected situations — a file that doesn't exist, a network timeout, a user entering the wrong input. Without error handling, your script crashes completely. With it, you handle the problem gracefully and keep going.

❌ crash without error handling 
FileNotFoundError: [Errno 2] No such file or directory: 'targets.txt'
PermissionError: [Errno 13] Permission denied: '/etc/shadow'
ConnectionRefusedError: [Errno 111] Connection refused
ValueError: invalid literal for int() with base 10: 'abc'
✅ Goal: Instead of crashing, your script should catch these errors, print a helpful message, and either retry, skip, or exit cleanly.
🛡️ try / except — The Core Pattern
▶️ try: — run the risky code here
🚨 except: — if an error occurs, run this instead
✅ else: — runs only if NO error occurred (optional)
🔒 finally: — ALWAYS runs, error or not (cleanup)
python – try/except basics 
# Without error handling — CRASHES if file missing
# f = open("missing.txt", "r")  ← crashes here

# With error handling — handles it gracefully
try:
    with open("targets.txt", "r") as f:
        data = f.read()
        print("File loaded!")
except FileNotFoundError:
    print("❌ Error: targets.txt not found.")
except PermissionError:
    print("❌ Error: No permission to read file.")
except Exception as e:
    print(f"❌ Unexpected error: {e}")
else:
    print("✅ File read successfully!")
finally:
    print("[*] Done — script continues...")
🎯 Best practice: Always catch specific exceptions first (FileNotFoundError, ValueError) before the generic Exception. Catching everything with bare except: hides bugs!
🖥️ os Module — Talk to Your Operating System

The os module lets Python interact with your operating system — list files, check paths, run commands, create directories.

python – os module essentials 
import os

# Current working directory
print(os.getcwd())            # /home/kali/tools

# List files in a directory
print(os.listdir("."))        # ['scanner.py', 'targets.txt']

# Check if a file/path exists
print(os.path.exists("/etc/passwd"))  # True

# Check if it's a file or directory
print(os.path.isfile("scanner.py"))   # True
print(os.path.isdir("/etc"))          # True

# Create a directory
os.makedirs("scan_results", exist_ok=True)

# Run a system command (use carefully!)
os.system("whoami")            # runs whoami in shell

# Get environment variables
print(os.environ.get("HOME"))  # /home/kali
⚙️ sys Module — Control Your Script

The sys module gives you control over the Python runtime — accept command-line arguments, exit cleanly, and more.

python – sys module 
import sys

# Exit the script cleanly
if len(sys.argv) < 2:
    print("Usage: python scanner.py <target_ip>")
    sys.exit(1)   # exit with error code 1

# sys.argv — command line arguments
# python scanner.py 192.168.1.1
# sys.argv[0] = "scanner.py"
# sys.argv[1] = "192.168.1.1"
target = sys.argv[1]
print(f"Scanning: {target}")

# Python version info
print(sys.version)   # 3.11.2
🔥 sys.argv is key for real tools! Every professional hacking tool accepts arguments from the command line. python scanner.py 10.0.0.1 22 — target and port passed right in. This is how Nmap, Nikto, and every CLI tool works.
🛡️ Mini Project — Robust File Scanner
python – robust_scanner.py 
import os, sys

def load_targets(filepath):
    if not os.path.exists(filepath):
        print(f"❌ File not found: {filepath}")
        sys.exit(1)
    try:
        with open(filepath, "r") as f:
            return [line.strip() for line in f if line.strip()]
    except PermissionError:
        print(f"❌ Permission denied: {filepath}")
        sys.exit(1)

def save_results(results, outfile):
    try:
        with open(outfile, "w") as f:
            for line in results:
                f.write(line + "\n")
        print(f"✅ Results saved → {outfile}")
    except Exception as e:
        print(f"❌ Could not save: {e}")

# Main logic
targets = load_targets("targets.txt")
results = []
print(f"[*] Loaded {len(targets)} targets\n")

for target in targets:
    result = f"[SCAN] {target} — checked"
    results.append(result)
    print(result)

save_results(results, "output.txt")
