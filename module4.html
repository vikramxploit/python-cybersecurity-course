⚙️ What is a Function?

A function is a named block of reusable code. Instead of writing the same logic 10 times, you write it once in a function and call it by name whenever you need it. This is called DRY — Don't Repeat Yourself.

python – defining and calling a function 
# Define a function with "def"
def greet_hacker():
    print("Welcome to the terminal, hacker!")
    print("Stay ethical. Stay legal.")

# Call the function — just write its name with ()
greet_hacker()
greet_hacker()   # call it again — no need to rewrite!
⬇ output 
Welcome to the terminal, hacker!
Stay ethical. Stay legal.
Welcome to the terminal, hacker!
Stay ethical. Stay legal.
📥 Parameters & Return Values

Parameters let you pass data INTO a function. return sends a result back OUT so you can use it elsewhere.

python – functions with parameters & return 
# Function with a parameter
def get_service(port):
    if port == 22:
        return "SSH"
    elif port == 80:
        return "HTTP"
    elif port == 443:
        return "HTTPS"
    else:
        return "Unknown"

# Call with different arguments
print(get_service(22))    # SSH
print(get_service(443))   # HTTPS
print(get_service(9999))  # Unknown

# Store the returned value
svc = get_service(80)
print(f"Port 80 runs: {svc}")  # Port 80 runs: HTTP

# Multiple parameters
def scan_report(ip, port):
    svc = get_service(port)
    print(f"[*] {ip}:{port} → {svc}")

scan_report("10.0.0.1", 22)
scan_report("10.0.0.2", 80)
⬇ output 
SSH
HTTPS
Unknown
Port 80 runs: HTTP
[*] 10.0.0.1:22 → SSH
[*] 10.0.0.2:80 → HTTP
💡 Default parameters: You can set a default value so the argument is optional:
def scan(ip, port=80): — if you call scan("10.0.0.1") it uses port 80 automatically.
📖 Dictionaries — Key → Value Lookup

A dictionary stores data as key: value pairs — like a real dictionary where you look up a word (key) to get its definition (value). Instant lookup — no looping needed.

21→"FTP"
22→"SSH"
80→"HTTP"
443→"HTTPS"
3306→"MySQL"
python – dictionaries 
# Create a dictionary { key: value }
services = {
    21:   "FTP",
    22:   "SSH",
    80:   "HTTP",
    443:  "HTTPS",
    3306: "MySQL"
}

# Look up a value by key
print(services[22])        # SSH
print(services[443])       # HTTPS

# .get() — safe lookup (no crash if key missing)
print(services.get(9999, "Unknown"))  # Unknown

# Add a new entry
services[8080] = "HTTP-Alt"

# Check if key exists
print(22 in services)      # True

# Loop through a dictionary
for port, svc in services.items():
    print(f"Port {port:5} → {svc}")
⚠️ Dict vs List: Lists use numeric indexes [0]. Dicts use named keys ["ssh"]. Use a dict when your data has meaningful labels — always use .get() to avoid crashes on missing keys.
🛡️ Mini Project — Service Scanner with Functions + Dicts
python – service_scanner.py 
# ===== Service Scanner =====

SERVICES = {
    21:"FTP", 22:"SSH", 23:"Telnet",
    25:"SMTP", 53:"DNS", 80:"HTTP",
    443:"HTTPS", 3306:"MySQL", 8080:"HTTP-Alt"
}
RISKY = [21, 23, 3306]

def identify(port):
    return SERVICES.get(port, "Unknown")

def risk_level(port):
    return "HIGH" if port in RISKY else "LOW"

def scan(ip, open_ports):
    print(f"\n{'='*45}")
    print(f"  🔍 TARGET: {ip}")
    print(f"{'='*45}")
    for port in open_ports:
        svc  = identify(port)
        risk = risk_level(port)
        icon = "⚠️ " if risk=="HIGH" else "✅"
        print(f"  {icon} {port:5} | {svc:10} | {risk}")
    print(f"{'='*45}\n")

# Run the scanner
scan("192.168.1.1", [22, 80, 21, 443, 3306])
