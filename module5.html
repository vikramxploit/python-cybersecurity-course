📖 Opening Files — The open() Function

Python's built-in open() function lets you read from and write to files. Always pair it with a mode — r, w, a, or x.

"r"
Read — open existing file
"w"
Write — overwrite (creates if missing)
"a"
Append — add to end of file
"x"
Create — fail if file already exists
💡 Always use the with keyword when opening files. It automatically closes the file when done — even if an error occurs. This prevents data corruption and memory leaks.
📥 Reading Files
python – reading a file 
# Method 1: read entire file as one string
with open("targets.txt", "r") as f:
    content = f.read()
    print(content)

# Method 2: read line by line (memory efficient!)
with open("targets.txt", "r") as f:
    for line in f:
        line = line.strip()   # remove \n newline
        print(f"Target: {line}")

# Method 3: read all lines into a list
with open("targets.txt", "r") as f:
    lines = f.readlines()

print(f"Total targets: {len(lines)}")
⚡ Always .strip() each line! Every line read from a file has a hidden \n (newline character) at the end. Without .strip(), "admin\n" won't match "admin" and your comparisons will silently fail.
📤 Writing Files — Logs & Reports
python – writing to files 
# Write mode — creates or OVERWRITES
with open("scan_results.txt", "w") as f:
    f.write("=== Scan Report ===\n")
    f.write("Target: 192.168.1.1\n")
    f.write("Open ports: 22, 80, 443\n")

# Append mode — adds to existing file
with open("scan_results.txt", "a") as f:
    f.write("Target: 10.0.0.5\n")
    f.write("Open ports: 21, 3306\n")

# Write multiple lines from a list
found_hosts = ["10.0.0.1", "10.0.0.5", "10.0.0.12"]
with open("live_hosts.txt", "w") as f:
    for host in found_hosts:
        f.write(host + "\n")
print("✅ Results saved to live_hosts.txt")
⚠️ w vs a mode: w (write) will completely erase the file before writing — be careful! Use a (append) when you want to keep adding to a log file without destroying previous entries.
🛡️ Mini Project — Wordlist Brute Force Simulator

This simulates reading a password wordlist and checking each one — exactly how real brute force tools work:

python – wordlist_attack.py 
# ===== Wordlist Attack Simulator =====
# (Educational / ethical use only!)

TARGET_PASSWORD = "s3cur3!"   # simulated target
WORDLIST        = "wordlist.txt"

def brute_force(wordlist_path, target):
    attempts = 0
    print(f"[*] Loading wordlist: {wordlist_path}")
    print(f"[*] Starting attack...\n")

    with open(wordlist_path, "r") as f:
        for line in f:
            password = line.strip()
            attempts += 1
            print(f"  Trying: {password:15}", end="")

            if password == target:
                print(" ✅ FOUND!")
                print(f"\n{'='*38}")
                print(f"  🔓 Password cracked: {password}")
                print(f"  Attempts needed  : {attempts}")
                print(f"{'='*38}")
                return True
            else:
                print(" ❌")

    print("\n[!] Password not found in wordlist.")
    return False

brute_force(WORDLIST, TARGET_PASSWORD)
⬇ output (with wordlist: admin, password, 123456, s3cur3!, root) 
[*] Loading wordlist: wordlist.txt
[*] Starting attack...

  Trying: admin           ❌
  Trying: password        ❌
  Trying: 123456          ❌
  Trying: s3cur3!         ✅ FOUND!

======================================
  🔓 Password cracked: s3cur3!
  Attempts needed  : 4
======================================
🔐 Why this matters: Real tools like Hydra and Medusa work exactly like this — they read from wordlists like rockyou.txt (14 million passwords) and try each one. Understanding this helps you build stronger defenses.
